ERKLÄRUNG DER DATEI: auswertung.php
Eine Anfänger-freundliche Erklärung der wichtigsten Code-Elemente
================================================================================

WAS MACHT DIESE DATEI?

Die Datei auswertung.php zeigt dem Teamchef eine Trainingsauswertung an.
Der Teamchef kann optional nach Trainingsziel und Zeitraum filtern.
Danach werden für jeden Fahrer des eigenen Teams Monatsstatistiken berechnet.

Die Datei besteht aus zwei großen Bereichen:

1. PROCESS-PHASE
   Hier werden Daten aus der Datenbank geladen und Formular-Eingaben verarbeitet.

2. RENDER-PHASE
   Hier wird das Formular und danach die Ergebnis-Tabelle im Browser angezeigt.


==== DATEI-HEADER ====
<?php
/*
 * Autor: Magdalena Hamm
 * Dieses Modul besteht aus zwei Teilen:
 * In der process-Phase werden Fahrer, Trainingsziele und bei Bedarf Statistiken geladen.
 * In der render-Phase werden Formular und Ergebnis-Tabelle ausgegeben.
 */

ERKLÄRUNG:
- "<?php" startet PHP-Code.
- "/* ... */" ist ein Kommentarblock.
- Kommentare werden vom Computer ignoriert, helfen aber Menschen beim Verstehen.
- Der Kommentar beschreibt direkt, wofür die Datei gedacht ist:
  Daten laden, Statistiken berechnen und Ergebnisse anzeigen.


==== SICHERHEITS-SCHUTZ ====
if (!defined('TEAMCHEF_DASHBOARD')) {
    header('Location: teamchef_dashboard.php');
    exit;
}

ERKLÄRUNG:
- "defined('TEAMCHEF_DASHBOARD')" prüft, ob die Konstante TEAMCHEF_DASHBOARD existiert.
- "!" bedeutet "nicht".
- Wenn die Konstante nicht existiert, wurde die Datei wahrscheinlich direkt aufgerufen.
- Dann wird der Benutzer mit header(...) zum Teamchef-Dashboard weitergeleitet.
- "exit;" beendet die Ausführung sofort.

ZWECK:
Diese Datei soll nur als Teil des Dashboards laufen.
Sie soll nicht direkt im Browser geöffnet werden.


==== PROCESS-PHASE STARTET ====
if (($dashboardPhase ?? '') === 'process') {

ERKLÄRUNG:
- "$dashboardPhase" sagt, in welcher Phase das Dashboard gerade ist.
- "process" bedeutet: Daten vorbereiten und Formularaktionen verarbeiten.
- "?? ''" bedeutet: Falls $dashboardPhase nicht existiert, verwende einen leeren Text.
- So entsteht keine Fehlermeldung, wenn die Variable einmal nicht gesetzt ist.


==== STANDARDWERTE SETZEN ====
$auswertungStatistiken = array();
$auswertungZiel = '';
$auswertungVon = '';
$auswertungBis = '';
$auswertungWurdeAngefordert = false;

ERKLÄRUNG:
- "$auswertungStatistiken" ist eine Liste für spätere Statistik-Ergebnisse.
- "$auswertungZiel" speichert das ausgewählte Trainingsziel.
- "$auswertungVon" speichert das Startdatum.
- "$auswertungBis" speichert das Enddatum.
- "$auswertungWurdeAngefordert" merkt sich, ob der Benutzer schon auf "Auswerten" geklickt hat.

WARUM IST DAS WICHTIG?
Diese Standardwerte sorgen dafür, dass das Formular auch beim ersten Laden der Seite funktioniert.
Ohne diese Werte könnten später undefinierte Variablen entstehen.


==== FAHRER DES TEAMS LADEN ====
$auswertungFahrerListe = array();
$fahrerStmt = mysqli_prepare($connection, 'SELECT Mitarbeiter_ID, Name FROM Fahrer WHERE Team = ? ORDER BY Name');

ERKLÄRUNG:
- "$auswertungFahrerListe" ist eine leere Liste.
- In diese Liste kommen alle Fahrer des aktuellen Teams.
- "mysqli_prepare(...)" bereitet einen SQL-Befehl sicher vor.
- "$connection" ist die Verbindung zur Datenbank.
- Der SQL-Befehl lautet:

SELECT Mitarbeiter_ID, Name FROM Fahrer WHERE Team = ? ORDER BY Name

SQL-BEFEHL ERKLÄRT:
- "SELECT Mitarbeiter_ID, Name" = Hole die Spalten Mitarbeiter_ID und Name.
- "FROM Fahrer" = Aus der Tabelle Fahrer.
- "WHERE Team = ?" = Nur Fahrer aus einem bestimmten Team.
- "?" ist ein Platzhalter.
- "ORDER BY Name" = Sortiere alphabetisch nach Name.


==== FAHRER-ABFRAGE AUSFÜHREN ====
if ($fahrerStmt) {
    mysqli_stmt_bind_param($fahrerStmt, 's', $teamRaw);
    mysqli_stmt_execute($fahrerStmt);
    mysqli_stmt_bind_result($fahrerStmt, $fahrerId, $fahrerName);

    while (mysqli_stmt_fetch($fahrerStmt)) {
        $auswertungFahrerListe[] = array('Mitarbeiter_ID' => $fahrerId, 'Name' => $fahrerName);
    }

    mysqli_stmt_close($fahrerStmt);
}

ERKLÄRUNG:
- "if ($fahrerStmt)" prüft, ob die Vorbereitung erfolgreich war.
- "mysqli_stmt_bind_param(...)" setzt den Platzhalter "?".
- "'s'" bedeutet: Der Wert ist ein String, also Text.
- "$teamRaw" ist der Teamname des aktuellen Teamchefs.
- "mysqli_stmt_execute(...)" führt die Abfrage aus.
- "mysqli_stmt_bind_result(...)" legt fest, in welche PHP-Variablen die Ergebnisse gespeichert werden.
- "while (mysqli_stmt_fetch(...))" läuft durch alle gefundenen Fahrer.
- Jeder Fahrer wird als Array in "$auswertungFahrerListe" gespeichert.
- "mysqli_stmt_close(...)" schließt die vorbereitete Abfrage.

BEISPIEL FÜR DIE LISTE:
[
  ['Mitarbeiter_ID' => 1, 'Name' => 'Anna'],
  ['Mitarbeiter_ID' => 2, 'Name' => 'Ben']
]


==== TRAININGSZIELE LADEN ====
$auswertungZiele = array();
$zielResult = mysqli_query($connection, 'SELECT Trainingsziel FROM Trainingsziel ORDER BY Trainingsziel');

ERKLÄRUNG:
- "$auswertungZiele" ist eine Liste für das Dropdown-Menü.
- "mysqli_query(...)" führt eine einfache SQL-Abfrage aus.
- Hier gibt es keine Benutzereingabe im SQL-Befehl, deshalb wird kein Platzhalter benötigt.

SQL-BEFEHL:
SELECT Trainingsziel FROM Trainingsziel ORDER BY Trainingsziel

BEDEUTUNG:
- Hole alle Trainingsziele aus der Tabelle Trainingsziel.
- Sortiere sie alphabetisch.


==== TRAININGSZIELE IN ARRAY SPEICHERN ====
if ($zielResult) {
    while ($row = mysqli_fetch_assoc($zielResult)) {
        $auswertungZiele[] = $row['Trainingsziel'];
    }
}

ERKLÄRUNG:
- "if ($zielResult)" prüft, ob die Abfrage erfolgreich war.
- "mysqli_fetch_assoc(...)" holt eine Ergebniszeile als assoziatives Array.
- Assoziativ bedeutet: Man greift über Spaltennamen zu, z.B. $row['Trainingsziel'].
- Jedes Trainingsziel wird an die Liste "$auswertungZiele" angehängt.


==== FORMULAR WURDE ABGESENDET? ====
if ($_SERVER['REQUEST_METHOD'] === 'POST' && $taskAction === 'auswertung_anzeigen') {

ERKLÄRUNG:
- "$_SERVER['REQUEST_METHOD']" sagt, mit welcher Methode die Seite aufgerufen wurde.
- "POST" bedeutet: Ein Formular wurde abgeschickt.
- "$taskAction === 'auswertung_anzeigen'" prüft, ob genau dieses Auswertungsformular gemeint ist.
- Beide Bedingungen müssen stimmen.

ZWECK:
Die Statistik soll erst berechnet werden, wenn der Benutzer auf "Auswerten" geklickt hat.


==== FILTER AUS DEM FORMULAR ÜBERNEHMEN ====
$auswertungWurdeAngefordert = true;
$auswertungZiel = post_value('auswertung_trainingsziel');
$auswertungVon = post_value('auswertung_von');
$auswertungBis = post_value('auswertung_bis');

ERKLÄRUNG:
- "$auswertungWurdeAngefordert = true" merkt: Ja, der Benutzer möchte eine Auswertung sehen.
- "post_value(...)" liest Werte aus dem POST-Formular.
- "$auswertungZiel" enthält das ausgewählte Trainingsziel oder leer für "Alle Ziele".
- "$auswertungVon" enthält das optionale Startdatum.
- "$auswertungBis" enthält das optionale Enddatum.


==== ZEITRAUM PRÜFEN ====
if ($auswertungVon !== '' && $auswertungBis !== '' && $auswertungVon > $auswertungBis) {
    $fehler = 'Das Von-Datum darf nicht nach dem Bis-Datum liegen.';
}

ERKLÄRUNG:
- Es wird nur geprüft, wenn beide Datumsfelder ausgefüllt sind.
- "$auswertungVon > $auswertungBis" bedeutet:
  Das Startdatum liegt nach dem Enddatum.
- Das wäre unlogisch, deshalb wird eine Fehlermeldung gesetzt.

BEISPIEL:
- Von: 2026-06-10
- Bis: 2026-06-01
- Das ist ungültig, weil der Zeitraum rückwärts läuft.


==== STATISTIK FÜR JEDEN FAHRER BERECHNEN ====
foreach ($auswertungFahrerListe as $fahrerEintrag) {
    $stats = new TrainingStats((int) $fahrerEintrag['Mitarbeiter_ID'], $auswertungZiel, $auswertungVon, $auswertungBis);

    if (!$stats->loadFromDatabase($connection, $teamRaw)) {
        $fehler = 'Auswertung konnte nicht geladen werden.';
        break;
    }

    $auswertungStatistiken[] = array(
        'fahrer' => $fahrerEintrag,
        'monate' => $stats->getMonatsStatistik(),
    );
}

ERKLÄRUNG:
- "foreach" läuft durch alle Fahrer des Teams.
- Für jeden Fahrer wird ein neues Objekt der Klasse "TrainingStats" erstellt.
- "(int) $fahrerEintrag['Mitarbeiter_ID']" wandelt die Fahrer-ID in eine ganze Zahl um.
- Dem Objekt werden Fahrer-ID, Trainingsziel, Von-Datum und Bis-Datum übergeben.

WAS MACHT TrainingStats?
- Die Klasse TrainingStats steht in TrainingStats.php.
- Sie lädt Trainingsdaten eines Fahrers aus der Datenbank.
- Sie gruppiert die Werte nach Monat.
- Sie berechnet Kennzahlen wie Summe, Durchschnitt, Minimum, Maximum, Median und Standardabweichung.

ERKLÄRUNG DER METHODEN:
- "$stats->loadFromDatabase($connection, $teamRaw)" lädt die Trainingsdaten.
- Wenn das Laden fehlschlägt, wird ein Fehler gesetzt und die Schleife abgebrochen.
- "$stats->getMonatsStatistik()" gibt die berechnete Monatsstatistik zurück.

ERGEBNIS:
In "$auswertungStatistiken" steht danach für jeden Fahrer:
- die Fahrerdaten
- die Statistikwerte pro Monat


==== RENDER-PHASE STARTET ====
if (($dashboardPhase ?? '') === 'render') {
?>

ERKLÄRUNG:
- "render" bedeutet: Jetzt wird HTML ausgegeben.
- "?>" beendet PHP vorübergehend.

==== TRAININGSZIEL-DROPDOWN ====
<label for="auswertung_trainingsziel">Trainingsziel:</label><br>
<select name="auswertung_trainingsziel" id="auswertung_trainingsziel">
    <option value="">Alle Ziele</option>
    <?php foreach ($auswertungZiele as $zielOption) { ?>
        <option value="<?php echo e($zielOption); ?>" <?php if ($auswertungZiel === $zielOption) echo 'selected'; ?>>

ERKLÄRUNG:
- "<label>" ist die Beschriftung für das Auswahlfeld.
- "<select>" ist ein Dropdown-Menü.
- Die erste Option hat value="" und bedeutet: Kein bestimmtes Ziel, also alle Ziele.
- "foreach ($auswertungZiele as $zielOption)" erstellt für jedes Trainingsziel eine Option.
- "echo e(...)" gibt Text sicher aus.
- "e(...)" schützt vor problematischen HTML-Zeichen.
- Wenn ein Ziel bereits ausgewählt wurde, wird "selected" ausgegeben.

ZWECK:
Nach dem Absenden bleibt die Auswahl sichtbar.
Der Benutzer sieht also, mit welchem Filter die Auswertung erstellt wurde.

<?php foreach ($auswertungStatistiken as $fahrerStatistik) { ?>
    <?php foreach ($fahrerStatistik['monate'] as $monat => $werte) { ?>
        <tr>
            <td><?php echo e($fahrerStatistik['fahrer']['Mitarbeiter_ID'] . ' - ' . $fahrerStatistik['fahrer']['Name']); ?></td>

- Die äußere foreach-Schleife läuft durch alle Fahrer.
- Die innere foreach-Schleife läuft durch alle Monate dieses Fahrers.

BEISPIEL:
Ein Fahrer hat Trainings im Januar und Februar.
Dann entstehen zwei Zeilen für diesen Fahrer:
- eine Zeile für 2026-01
- eine Zeile für 2026-02


==== ZAHLEN FORMATIEREN ====
number_format($werte['summe'], 2, ',', '.')

ERKLÄRUNG:
- "number_format(...)" formatiert eine Zahl für die Anzeige.
- Der zweite Parameter "2" bedeutet: zwei Nachkommastellen.
- Das Komma "," wird als Dezimaltrennzeichen genutzt.
- Der Punkt "." wird als Tausendertrennzeichen genutzt.

BEISPIEL:
1234.5 wird angezeigt als:

================================================================================
ZUSAMMENFASSUNG DER GESAMTEN DATEI
================================================================================

1. Die Datei schützt sich vor direktem Aufruf.
2. In der process-Phase werden alle Fahrer des aktuellen Teams geladen.
3. Außerdem werden alle Trainingsziele für das Dropdown geladen.
4. Wenn der Benutzer das Formular abschickt, werden die Filter übernommen.
5. Der Zeitraum wird geprüft.
6. Für jeden Fahrer wird ein TrainingStats-Objekt erstellt.
7. TrainingStats lädt die Trainingsdaten und berechnet Monatsstatistiken.
8. In der render-Phase wird das Formular angezeigt.
9. Nach erfolgreicher Auswertung wird eine Tabelle mit Statistikwerten angezeigt.
10. Wenn keine passenden Trainings vorhanden sind, erscheint "Keine Trainings gefunden."


DATENFLUSS:

1. Teamchef öffnet den Bereich "Auswertung".
2. Die Seite lädt Fahrer und Trainingsziele aus der Datenbank.
3. Teamchef wählt optional Trainingsziel, Von-Datum und Bis-Datum.
4. Teamchef klickt auf "Auswerten".
5. Die Seite prüft die Eingaben.
6. Für jeden Fahrer werden Trainingsdaten geladen.
7. Die Werte werden pro Monat gruppiert.
8. Statistikwerte werden berechnet.
9. Die Tabelle wird im Browser ausgegeben.


SICHERHEITS-FEATURES:

1. Die Datei darf nur über das Teamchef-Dashboard laufen.
2. Fahrer werden nur für das aktuelle Team geladen.
3. SQL mit Team-Filter nutzt Prepared Statements.
4. Formularwerte werden über post_value(...) gelesen.
5. HTML-Ausgaben laufen über e(...), damit Sonderzeichen sicher angezeigt werden.
6. Der Zeitraum wird geprüft, damit keine unlogische Auswertung entsteht.


WICHTIGE FUNKTIONEN UND VARIABLEN:

- $connection = Verbindung zur Datenbank.
- $teamRaw = aktuelles Team des Teamchefs.
- $auswertungFahrerListe = Liste aller Fahrer des Teams.
- $auswertungZiele = Liste aller Trainingsziele.
- $auswertungStatistiken = fertige Statistikdaten für die Ausgabe.
- mysqli_prepare() = SQL-Befehl sicher vorbereiten.
- mysqli_stmt_bind_param() = Werte sicher an Platzhalter binden.
- mysqli_stmt_execute() = vorbereiteten SQL-Befehl ausführen.
- mysqli_fetch_assoc() = Datenbankzeile als Array lesen.
- TrainingStats = Klasse für Statistikberechnung.
- number_format() = Zahl schön lesbar formatieren.
- foreach = Schleife über Listen.
